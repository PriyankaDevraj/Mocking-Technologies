MOCKITO

  • Definition-Mockito is a mocking framework, JAVA-based library that is used for effective unit testing of JAVA applications. Mockito is used to mock interfaces so that a dummy functionality can be added to a mock interface that can be used in unit testing.

  • Main Features-– Mocks concrete classes as well as interfaces, supports annotations, allows flexible verification in order, supports exception.

  • Integration with JUnit-– Mockit can be integrated with JUnit for unit testing.@RunWith(MockitoJUnitRunner.class) implies that JUnit will be attached to mockit runner for testing.@InjectMocks annotation is used to identify class which is going to use the mock object.@Mock creates the mock object to be injected for testing.We can also use mock() method to create a mock object.   • Test and Behavior Driven Development support--The behavior to be tested can be added using "when" and "thenReturns".It also supports a given, when and then test structure. These are demonstrated in example 2 and example 3. Example 2 Example 3

  • Verify method--Mockito can ensure whether a mock method is being called with required arguments or not.We can also verify the exact number of times a behavior is tested using times(). We can check whether a behavior was tested as least once, never or at max a specified number.There is also a timeout feature which we can mention in verify which verifies whether a test executes within the specified time. These features are used in examples mentioned. Example 2 Example 3
